Kamu adalah AI Software Engineer senior yang sangat kuat di BACKEND (Golang gRPC, arsitektur bersih, DB), FRONTEND Admin Dashboard (Next.js), dan MOBILE (Flutter). Kamu harus membangun sistem end-to-end, production-grade, dengan standar keamanan, audit trail, dan dokumentasi API.

TARGET SISTEM
Buat “Portal Bapperida Kalimantan Tengah” dengan arsitektur dual-interface:
1) Admin Panel Web (Next.js) untuk manajemen modul (RBAC)
2) Mobile App Flutter yang konsumsi API publik & user-auth

Modul utama:
A. Domain BAPPEDA (Badan Perencanaan Pembangunan) menangani:
   - News (CMS berita)
   - Dokumen (PPID documents: Jenis, Kategori, Tipe)
   - Banner (jadwal tayang + tracking view/click)
   - Menus (menu dinamis untuk mobile)
   - Pages statis (optional jika sempat)
B. Domain RIDA (Riset dan Inovasi Daerah) menangani:
   - Izin Penelitian (pengajuan -> review -> revisi(optional) -> approved -> generate surat PDF -> kirim via email/WA)
   - Survei Kepuasan (IKM) setelah layanan
   - Laporan Akhir Penelitian & Kotak Saran

TECH STACK (HARUS VERSI TERBARU SAAT EKSEKUSI)
- Backend: Golang (gunakan versi stable terbaru di Replit), gRPC, PostgreSQL, Redis (optional), object storage local filesystem (untuk Replit), JWT auth.
- Expose REST JSON untuk konsumsi Next.js & Flutter menggunakan grpc-gateway (jadi internal gRPC, eksternal REST).
- Admin Web: Next.js versi terbaru (App Router), TypeScript, Tailwind, React Query, Zod, SSR/CSR sesuai kebutuhan, UI yang rapi dan cepat.
- Mobile: Flutter versi terbaru stable, Dio/Retrofit, Riverpod/BLoC (pilih satu dan konsisten), Freezed/JsonSerializable untuk model.

9) Modul IZIN PENELITIAN (RIDA) – workflow WAJIB:
   - Public/mobile: form pengajuan “Permohonan Surat Izin Penelitian”
   - Data fields (ikuti yang dijabarkan):
     email, nama lengkap, email aktif, NIM/NIK, tempat lahir, unit kerja/tim, asal lembaga, no telp whatsapp, kewarganegaraan (WNI/WNA),
     lokasi penelitian spesifik, durasi penelitian, judul penelitian,
     jabatan penandatangan surat pengantar, nomor surat, tanggal surat,
     upload PDF: kartu identitas, surat pengantar, proposal penelitian, bukti follow sosial media (anggap pdf), bukti isi survei kepuasan (pdf),
     agreement: bersedia menyampaikan laporan akhir ke BAPPERIDA bidang RIDA.
   - Validasi:
     pdf max 1MB per file (sesuai form), uppercase enforcement untuk field tertentu (kecuali email),
     jam layanan: proses hanya hari kerja Senin-Jumat 08:00-15:00 WIB, estimasi 2-5 hari kerja (simpan sebagai info/metadata; jangan hard-block submit tapi tandai SLA & jam operasional).
   - Status:
     submitted -> in_review -> revision_requested(optional) -> approved -> generated_letter -> sent(optional) atau rejected
   - Audit trail ganda:
     1) research_permit_status_histories (setiap perubahan status)
     2) activity log (minimal table audit_logs sendiri atau pakai log table)
   - Nomor permohonan otomatis: format BAPPERIDA-RID-YYYY-000001
     Harus race-condition safe: gunakan DB transaction + row lock pada request_sequences.
   - Generate Surat:
     letter_templates (HTML template dengan placeholder)
     generated_letters (path file, meta)
     PDF generator di Go (pakai library yang reliable atau html->pdf via headless chrome jika paling stabil di Replit; jelaskan pilihan).
   - Pengiriman:
     implement email sender (SMTP env) minimal; WhatsApp cukup disiapkan “stub” (webhook placeholder).
   - Endpoint user:
     GET /research-permits/me list permohonan saya
     GET /research-permits/{id} detail + URL surat bila sudah ada
   - Admin:
     list permohonan, detail, update status, request revision note, approve & generate letter, resend email.

AUTH & RBAC
- Admin Panel: RBAC roles: super_admin, admin_bpp, admin_rida
  super_admin: full access
  admin_bpp: akses modul News, Dokumen, Banner, Menus, Pages
  admin_rida: akses modul Izin Penelitian, Survei, Laporan Akhir, Template Surat
- Mobile user: role user (pemohon) untuk izin penelitian dan konsumsi konten.
- Implement JWT untuk mobile, session tidak wajib (karena Next.js juga bisa pakai JWT cookie httpOnly).
- Pastikan middleware RBAC ada di gateway REST.

API DESIGN
- gRPC untuk internal service-to-service dan definisi kontrak.
- Expose REST JSON via grpc-gateway:
  /api/v1/public/* untuk publik (news list, banner active, docs list, menu)
  /api/v1/auth/* register/login
  /api/v1/admin/* khusus admin (RBAC)
  /api/v1/user/* khusus user login (research permits me, upload final report, survey)
